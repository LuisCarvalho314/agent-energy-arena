# 05: Three v1 shipped scenarios + per-scenario tests

Status: ready-for-agent

## Parent

`.scratch/open-source-arena/PRD.md`

## What to build

Add a top-level `scenarios/` package with one Python file per shipped scenario. Each class declares its seed as a class attribute. v1 ships three:

- `scenarios/baseline.py` — `Baseline(NullScenario)` on seed 42. Kept as a class so the CLI surface is uniform.
- `scenarios/grid_stress.py` — Combines sustained low-wind weeks (weather override clipping wind speed to a low value over a date range) with a heatwave cluster (named heatwave events injected on specific days with chosen durations).
- `scenarios/economy_stress.py` — Combines a fuel-price shock (mutate per-fuel `fuel_costs` dict on world state), a crude-price collapse (mutate `crude_price` on world state), and a regulatory tightening (injected named event on a chosen day with a chosen duration).

Each scenario's `apply` is deterministic given `(world, day)` and consumes no random numbers. Each scenario appends a trace entry to `state.scenario_trace` when an effect fires, so the recorder shows what happened.

The agent has latitude to choose plausible numeric values for the v1 stress profiles — start/end days, wind-speed clip values, heatwave durations, fuel-cost multipliers, crude-price collapse magnitude, regulatory-event duration. Aim for stresses that are observably hard but not unwinnable on the baseline seed; the maintainer will tune in review. Record each chosen value in the scenario class as a named class attribute (not a magic number inside `apply`) so the tuning surface is obvious.

Per-scenario tests drive a few simulated days and assert the documented overrides/events fire on the documented days and clear on the documented days.

## Acceptance criteria

- [ ] `scenarios/baseline.py`, `scenarios/grid_stress.py`, `scenarios/economy_stress.py` each define a `Scenario` subclass with an `apply` method and a declared seed attribute.

- [ ] Each scenario can be loaded by dotted path via `load_scenario("scenarios.grid_stress")`.
- [ ] A short test per scenario asserts effects fire and clear on the documented days.
- [ ] No scenario consumes random numbers.

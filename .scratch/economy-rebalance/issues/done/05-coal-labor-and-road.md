# Coal labor footprint + road requirement

Status: ready-for-agent
Category: enhancement

## Parent

[PRD: Economy & balance pass](../PRD.md)

## What to build

Make coal plants feel like logistically heavy industrial assets rather than free drop-ins:

- Increase coal `jobs` from 8 to 30 so scaling coal requires growing the labor pool and the housing/jobs ratio matters for power decisions.
- Set coal `requires_road` to True so coal placement obeys the existing road-adjacency rule and rejects with `no_road_adjacency` when missing.
- Update the coal catalog description to reflect both constraints (catalog description is the in-game tooltip and the single source of truth for placement rules).

Coal capex, OPEX, fuel costs, and CO2 intensity are unchanged.

## Acceptance criteria

- [ ] Coal catalog entry has `jobs = 30` and `requires_road = True`.
- [ ] Coal catalog description mentions both the 30-worker labor requirement and the road requirement.
- [ ] Placing a coal plant without road adjacency fails the build with the existing `no_road_adjacency` error code.
- [ ] Coal output continues to scale by `workforce.efficiency` (no wage/payroll mechanic introduced).
- [ ] Existing tests that pin coal jobs or road behavior are re-anchored and pass.
- [ ] `make check` passes.

## Agent Brief

**Category:** enhancement
**Summary:** Make coal plants feel logistically heavy: bump labor footprint to 30 and require road adjacency.

**Current behavior:**
Coal plants employ 8 workers and have no road-adjacency requirement. Their fuel pipeline is invisible, the labor cost is trivial, and there is no logistical reason not to spam them.

**Desired behavior:**
Coal plants require 30 workers (output continues to scale by workforce efficiency) and require road adjacency at build time. A coal placement without an adjacent road is rejected with the existing `no_road_adjacency` structured error code. Capex, OPEX, fuel cost, and CO2 intensity are unchanged. The coal entry's catalog description reflects both the 30-worker requirement and the road requirement.

**Key interfaces:**
- The coal entry in the tile catalog — `jobs = 30`, `requires_road = True`, description string mentions both constraints.
- The existing road-adjacency build validator — coal now flows through it the same way other road-requiring tiles do; no new validator logic.
- Existing tests pinning coal jobs or road behavior — re-anchor.

**Acceptance criteria:**
- [ ] Coal catalog entry has `jobs = 30` and `requires_road = True`.
- [ ] Coal catalog description mentions both the 30-worker and road requirements.

**Out of scope:**
- Per-worker wages or payroll mechanics (labor is a scarcity lever, not a cost lever).

# Event probabilities doubled

Status: ready-for-agent
Category: enhancement

## Parent

[PRD: Economy & balance pass](../PRD.md)

## What to build

Double the daily probability constants for every random event so that resilience and event-readiness become real strategic concerns in a single play session. The per-event "skipped if already active" logic, the `regulatory_tightening` cap of 3, and the RNG draw ordering are all unchanged.

New constants:
- `HEATWAVE_PROB` 0.003 → 0.006
- `FUEL_PRICE_SHOCK_PROB` 0.002 → 0.004
- `DEMAND_SURPRISE_PROB` 0.003 → 0.006
- `PLANT_FAILURE_PROB` gas: 0.0014 → 0.0028
- `PLANT_FAILURE_PROB` coal: 0.0006 → 0.0012
- `REGULATORY_TIGHTENING_PROB` 0.001 → 0.002

New values:
- Heatwave daily probability: 0.003 → 0.006
- Fuel-price-shock daily probability: 0.002 → 0.004
- Demand-surprise daily probability: 0.003 → 0.006
- Plant-failure daily probability, gas: 0.0014 → 0.0028
- Plant-failure daily probability, coal: 0.0006 → 0.0012
- Regulatory-tightening daily probability: 0.001 → 0.002

**Key interfaces:**
- The probability constants in the events module — values updated as above.
- The "already-active" guard around every event type — behavior unchanged (no double-stacking).
- The regulatory-tightening cumulative cap — still 3.
- The order of RNG draws within the daily event tick — unchanged.

**Acceptance criteria:**
- [ ] Each probability constant matches the doubled value listed above.
